# Code of Conduct

Agent Bus is a project for connecting AI tools, remote assistant nodes, and model gateways. That only works if contributors can ask questions, disagree, and debug sensitive infrastructure without being embarrassed or pushed around.

## Our Standard

We expect contributors to:

- Be welcoming to new operators, adapter authors, and first-time open source contributors.
- Critique ideas, code, security posture, and product tradeoffs without attacking people.
- Keep private credentials, logs, prompts, IP addresses, domains, and machine details out of public issues and pull requests.
- Make room for different deployment realities: Windows, Linux, macOS, Docker, private networks, low-resource machines, and no-model offline setups are all valid.
- Assume good intent while still naming concrete risks clearly.

Unacceptable behavior includes harassment, insults, threats, sexualized attention, repeated bad-faith disruption, publishing private information, or pressuring someone to reveal credentials, private logs, or infrastructure details.

## Scope

This code applies in GitHub issues, pull requests, discussions, release comments, community chat, and any project space where Agent Bus work is being coordinated.

## Reporting

For public-safe concerns, open an issue with only the minimum details needed. For security-sensitive or private details, follow `SECURITY.md` and avoid posting secrets, exploit details, private prompts, or runtime logs publicly.

Maintainers may edit or remove comments, close threads, block accounts, or pause a discussion when behavior makes collaboration unsafe or unproductive.
